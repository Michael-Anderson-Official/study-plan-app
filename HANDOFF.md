# HANDOFF.md

最終更新: 2026-07-06

## アプリの目的

1週間の学習計画を管理する静的Webアプリです。各日の各教材列に、上半分は予習欄、下半分は復習欄として記録できます。教材リストを追加し、教材詳細から単位数・目標日・1日あたり単位数・目次を設定して、カレンダーへ予定を割り当てます。

## 現在の実装状態

コード上確認できた事実:

- 主要ファイルは `index.html` のみ。
- 外部ライブラリなし。HTML、CSS、JavaScriptがすべてインライン。
- データ保存は `localStorage`。
- セル保存キーは `YYYY-MM-DD-subN-prep` / `YYYY-MM-DD-subN-review`。
- 週の振り返り保存キーは `weekly-summary-YYYY-MM-DD`。
- 教材リスト保存キーは `materials-list`。
- 教材詳細保存キーは `material-details`。
- 週始まり設定保存キーは `weekStart`。
- GitHub Pagesで `main` ブランチから公開されている。

## 直近で修正した内容

コード上確認できた事実:

- PC版で教材追加ボタンが動かない問題への対策として、`materials-list` と `material-details` の読み込み時に、JSONパース後の値が配列か確認するようにした。
- 教材追加時に `materials-list` だけでなく `material-details` も保存するようにした。
- カレンダー上部に教材名が表示されるため、予定セル内には教材名を付けず、単位番号や目次項目だけを書き込むようにした。
- 「一週間をリセット」は、現在週のセル内容だけを消すようにした。
- 「一週間をリセット」では、教材リスト、教材詳細、ヘッダー、週の振り返りを消さない。
- 教材詳細パネル内に「削除」ボタンを追加した。
- 教材削除は `materialsList` と `materialsDetails` から該当教材を削除し、`localStorage` に保存し直す。
- 教材削除後、詳細パネルは閉じる。
- 教材削除後、カレンダーにすでに記入済みの予定は残る。

## 指定トピックの現在状態

### PC版で教材追加ボタンが動かない問題

コード上確認できた事実:

- `loadMaterials()` で `materials-list` と `material-details` を `JSON.parse` したあと、`Array.isArray` で配列確認している。
- 教材追加時は `materialsList.push(name)` と `materialsDetails.push(null)` の後、両方を `localStorage` に保存している。

未確認事項:

- すべての既存ユーザー環境で壊れた保存データから復旧できるかは未確認。

### 今日ハイライトの日本時間判定

コード上確認できた事実:

- `toLocalISO(date)` は `getFullYear()`、`getMonth()`、`getDate()` を使い、ローカル日付で `YYYY-MM-DD` を作る。
- `parseISOToDate(iso)` は `new Date(y, m, d)` でローカル日付として復元する。
- 今日ハイライトはブラウザのローカル日付を `toLocalISO` に変換し、行の日付ISOと比較している。

未確認事項:

- 端末タイムゾーンが日本時間以外の場合の表示は未確認。

### 月曜始まり・日曜始まり設定

コード上確認できた事実:

- `weekStartSelect` の値は `localStorage.setItem('weekStart', selected)` で保存される。
- 値は `monday` または `sunday`。
- 設定変更時は現在日付から週開始日を再計算し、`window.currentWeekStartISO` を更新する。
- ページ読み込み時にも保存済み `weekStart` を読んで週開始日を計算する。

未確認事項:

- 週始まり変更時に、すべての既存入力が期待通り別週キーとして扱われるかは網羅未確認。

### 今週に戻るボタン

コード上確認できた事実:

- `currentWeekBtn` が存在し、表示テキストは `今週へ`。
- クリック時は現在日付と `weekStart` 設定から週開始日を再計算し、`window.currentWeekStartISO` を更新する。
- 更新後に `updateWeekUI()` と `updateCellsForCurrentWeek()` を呼ぶ。

未確認事項:

- PC幅・スマホ幅の両方での手動クリック確認は未確認。

## 既知のバグ・注意点

コード上確認できた事実:

- 祝日は `holidays2026` の固定リストのみ。
- 教材削除しても、カレンダーにすでに書き込まれた予定は消えない。
- 教材列は最大7教材分のヘッダーに表示される設計。8件目以降の教材追加時のUIは十分に整理されていない可能性がある。
- 予定セル内に教材名を入れないため、後から「特定教材の予定だけ削除」は安全に判定できない。

未確認事項:

- `contenteditable` への入力保存が全モバイルブラウザで安定するか。
- ローカル日付の境界時刻で今日ハイライトが日本時間として期待通りになるか。
